[COMMAND]: gobuster
[IN USE]: gobuster dir -u http://[IP adress] -w /usr/share/wordlists/dirb/common.txt
[DEFINITION]: gobuster is a tool for brute force working with directory's and files on web servers finding hidden web 
	      pages or files.
[ADDITIONAL]: -u = URL adress
	      -w = file with lists of words that will try


[EXAMPLES OF USE]:
# Basic terminal script
gobuster dir -u http://10.10.x.x -w /usr/share/wordlists/dirb/common.txt

# With extentions files
gobuster dir -u http://10.10.x.x -w common.txt -x php,html,txt

# With status code
gobuster dir -u http://10.10.x.x -w common.txt -s 200,301,302

# Working with more threads (faster)
gobuster dir -u http://10.10.x.x -w common.txt -t 50

# Scanning for DNS server
gobuster dns -d cilj.com -w subdomains.txt

[COMMAND]: enum4linux
[IN USE]: enum4linux [IP adress]
[DEFINITION]: tool throught SMB protocol finding information on Linux or Windows machines.
[ADDITIONAL]: /

[EXAMPLES]:

# Basic scanning
enum4linux 10.112.182.26

# Only users
enum4linux -U 10.112.182.26

# Only groups
enum4linux -G 10.112.182.26

# Shared folders
enum4linux -S 10.112.182.26

# OS information
enum4linux -o 10.112.182.26

# Password policy
enum4linux -P 10.112.182.26

[ADDITIONAL DEFINITION]: SMB protocol is a shared resource inside of the network. (Files, printers, password policy, folders, Information of the system).

[COMMAND]: hydra
[IN USE]: hydra -l [name of the user] -P /usr/share/wordlists/rockyou.txt ssh://[IP adress]
[DEFINITION]: hydra is a brute force tool for services that request authentication.
[ADDITIONAL]: -l = one username
	      -P = list of passwords

[EXAMPLES]:

# SSH
hydra -l jan -P /usr/share/wordlists/rockyou.txt ssh://10.112.182.26

# FTP
hydra -l jan -P /usr/share/wordlists/rockyou.txt ftp://10.112.182.26

# HTTP login form
hydra -l admin -P /usr/share/wordlists/rockyou.txt http-post-form://10.112.182.26

# RDP (Windows remote desktop)
hydra -l admin -P /usr/share/wordlists/rockyou.txt rdp://10.112.182.26

[COMMAND]: scp 
[IN USE]: scp file.txt user@192.168.1.10:/home/user/
[DEFINITION]: scp is a command that copy file between two copmuters using SSH protocol. 
[ADDITIONAL]: -r = copy folder
	      -P = specific port
	      -i = SSH key
	      -v = debug info 

[EXAMPLES]:

# from remote to local 
scp korisnik@192.168.1.10:/home/user/fajl.txt /home/user/Desktop/

# one file
scp dokument.txt user@192.168.1.10:/home/user/

# more files
scp fajl1.txt fajl2.txt user@192.168.1.10:/home/user/

# whole folder
scp -r /home/user/projekat user@192.168.1.10:/var/www/

# different port
scp -P 2222 dokument.txt user@192.168.1.10:/home/user/

# specified SSH key in use
scp -i key.pem file.txt user@192.168.1.10:/tmp/

[COMMAND]: john
[IN USE]: john file.txt --wordlist=/usr/share/wordlists/rockyou.txt
[DEFINITION]: john is a tool for brute forcing passwords using hash, trying every different combitnation from --wordlist
 text file. [HASH in the file.txt -> john -> rockyou.txt (password -> hash) -> comparison -> plain text password] 
[ADDITIONAL]: --format=md5 sha256 ntlm bcrypt  = type of format
	      --list = list of formats


[EXAMPLES]:

# Dictionary attack (wordlist)
john --wordlist=rockyou.txt hash.txt

# Brute force (every combo)
john --incremental hash.txt

# Rule based (multipy wordlist, abc123 → Abc123!)
john --wordlist=rockyou.txt --rules hash.txt

# Mask attack (we know part of the password)
john --mask=?u?l?l?l?d?d hash.txt
